## Campaign Archiving System

### How It Works:

**Automatic Archiving:**
1. Campaign with **30 consecutive days of no activity** is flagged
2. **Day 25**: Warning email sent to DM and players
   - "Your campaign '[Name]' will be archived in 5 days due to inactivity"
   - One-click link to "Keep Active"
3. **Day 30**: Campaign auto-archives
   - Moves to "Archived Campaigns" section
   - All data preserved (fully restorable)
   - Stops counting toward active campaign limits
   - Reduces database load

**Manual Archiving:**
- DM can manually archive any campaign
- Players can archive campaigns they're in (for themselves only)

**Restoration:**
- One-click "Restore Campaign" button
- Takes 5-30 seconds to reload
- All data intact (characters, sessions, state)
- Seamless return to gameplay

**Tier-Specific Archiving:**

| Tier | Archive After | Extended Grace | Manual Archive |
|------|---------------|----------------|----------------|
| **Free** | 30 days | âŒ | âœ… |
| **Creator** | 45 days | âœ… +15 days | âœ… |
| **Master** | Never | âœ… Infinite | âœ… |

**Benefits:**
- **For Users**: Never lose campaign data, clean active campaign list
- **For Us**: Reduced database costs, improved performance, better UX
- **For System**: Distinguish "paused" from "abandoned" campaigns

**Storage:**
```
Active Campaign: Hot storage (PostgreSQL + Redis)
Archived Campaign: Cold storage (S3/compressed JSON)
  - 90% cost reduction
  - Quick restoration (<30 seconds)
  - Full data preservation
```

### Archive Function:

```python
async def archive_campaign(campaign_id: str):
    """
    Archive a campaign to cold storage
    """
    # 1. Get full campaign data
    campaign = await get_full_campaign_data(campaign_id)
    
    # 2. Serialize to JSON
    campaign_json = serialize_campaign(campaign)
    
    # 3. Compress and upload to S3
    compressed = gzip.compress(campaign_json.encode())
    s3_key = f"archived_campaigns/{campaign_id}.json.gz"
    await s3.upload(s3_key, compressed)
    
    # 4. Update database
    await db.execute("""
        UPDATE campaigns 
        SET status = 'archived',
            archived_at = NOW(),
            archived_s3_key = $2
        WHERE id = $1
    """, campaign_id, s3_key)
    
    # 5. Clear hot storage (optional - keep metadata)
    # await clear_session_cache(campaign_id)
    
    # 6. Notify DM and players
    await send_archived_notification(campaign)
    
    logger.info(f"Campaign {campaign_id} archived to {s3_key}")
```

### Restore Function:

```python
async def restore_campaign(campaign_id: str, user_id: str):
    """
    Restore archived campaign to active status
    """
    # 1. Check permissions
    campaign = await db.fetchrow("""
        SELECT * FROM campaigns WHERE id = $1
    """, campaign_id)
    
    if campaign['dm_user_id'] != user_id:
        raise PermissionError("Only DM can restore campaign")
    
    if campaign['status'] != 'archived':
        raise ValueError("Campaign is not archived")
    
    # 2. Download from S3
    s3_key = campaign['archived_s3_key']
    compressed_data = await s3.download(s3_key)
    
    # 3. Decompress and parse
    campaign_json = gzip.decompress(compressed_data).decode()
    campaign_data = json.loads(campaign_json)
    
    # 4. Restore to database (if needed)
    # Most data should still be in DB, just update status
    
    # 5. Update status
    await db.execute("""
        UPDATE campaigns 
        SET status = 'active',
            last_activity = NOW(),
            archive_notification_sent = FALSE
        WHERE id = $1
    """, campaign_id)
    
    # 6. Warm up caches
    await warm_campaign_cache(campaign_id)
    
    logger.info(f"Campaign {campaign_id} restored by user {user_id}")
    
    return campaign_data
```
